# Charitable – Розподілена вебплатформа для автоматизованого збору благодійних пожертвувань

## Опис проєкту

**Charitable** — це сучасне розподілене цифрове середовище, спроєктоване для організації прозорого, зручного та безпечного збору благодійних пожертвувань. Платформа дозволяє користувачам ініціювати цільові збори, підтримувати наявні волонтерські кампанії, відстежувати персональну статистику внесків, а також підвищувати рівень суспільної залученості через механізми ігрофікації (гейміфікації). 

Головною особливістю системи є інтеграція інтелектуального модуля біометричної верифікації осіб розпорядників коштів для запобігання шахрайству та крадіжці цифрової тотожності.

Проєкт розроблено в межах **магістерської кваліфікаційної роботи** студента Факультету прикладної математики та інформатики Львівського національного університету імені Івана Франка.

## Основні можливості

- **Рольова модель безпеки:** Розмежування прав доступу між ролями `PHILANTHROPIST` (доброчинець) та `HELP_SEEKER` (Verified організатор зборів) за допомогою Spring Security.
- **Автоматизована ШІ-верифікація (KYC):** Нейромережевий аналіз тотожності особи шляхом порівняння фізіономічних ознак обличчя на актуальному селфі-знімку та наданому документі.
- **Управління благодійними проєктами:** Створення та адміністрування зборів із гнучким налаштуванням описів, категорій, цільових сум та візуального контенту.
- **Безпечний транзакційний процесинг:** Інтеграція з платіжним шлюзом **LiqPay** для здійснення безготівкових переказів у режимі реального часу.
- **Інтерактивна аналітика:** Візуалізація фінансового прогресу та динаміки зборів за допомогою бібліотеки **Google Charts**.
- **Елементи ігрофікації:** Динамічна система досягнень, віртуальних відзнак та нагород за високу соціальну активність.
- **Локалізація та адаптивність:** Повна багатомовна підтримка (**українська / англійська**) та ергономічний адаптивний дизайн із можливістю перемикання кольорового оформлення (**світла / темна теми**).
- **Хмарна інфраструктура:** Бездискова потокова обробка файлів через **Dropzone.js** із захищеним збереженням медіаконтенту в об'єктному сховищі **Amazon S3 (AWS)**.

## Технологічний стек

### Головний сервер оркестрації (Java)
- **Мова програмування:** Java 17
- **Каркас:** Spring Boot, Spring Security, Spring Data JPA
- **Керування залежностями та збирання:** Apache Maven
- **База даних:** PostgreSQL
- **Зовнішні інтеграції:** AWS SDK (Amazon S3), LiqPay API

### Обчислювальний ШІ-мікросервіс (Python)
- **Вебкаркас:** Python Flask (асинхронна багатопотокова обробка)
- **Глибоке машинне навчання:** PyTorch
- **Нейромережеві архітектури:** MTCNN (детектування облич), FaceNet / InceptionResnetV1 (екстракція біометричних ознак)
- **Математичні обчислення:** SciPy, NumPy (розрахунок косинусної схожості)

### Клієнтська частина
- Серверний шаблонізатор Apache FreeMarker (`.ftlh`)
- HTML5, CSS3 (Flexbox, Grid, Media Queries)
- JavaScript, jQuery, Dropzone.js

## Архітектура системи

Платформа базується на децентралізованій **мікросервісній архітектурі** з використанням патерна **MVC** на головному вузлі:

- **Model:** Сутності JPA, що відображають структуру реляційної СУБД (Users, Donations, Requests) та забезпечують цілісність даних (ACID).
- **View:** Динамічні шаблони FreeMarker, які відокремлюють презентаційний шар від бізнес-логіки.
- **Controller / Proxy Gateway:** Обробка HTTP-запитів клієнта та маршрутизація Multipart-потоків байтів через захищений TLS-тунель **ngrok** до ШІ-модуля без проміжного збереження на диск.
